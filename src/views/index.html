Como acessar o webservice do ViaDDD?

Para ter a acesso ao serviço basta você informar o DDD no formato de {3} dígitos, por exemplo: "013". Após informado corretamente você terá o retorno em formato "JSON".

Exemplo de pesquisa por DDD: viaddd.com.br/api/013



Formato de Retorno



Formato válido

Veja exemplo de acesso ao webservice e o seu retorno:

JSON


URL: viacep.com.br/ws/01001000/json/
UNICODE: viacep.com.br/ws/01001000/json/ 
            
              {
                "DDD":13,
                "Municipios":["Peruibe", "São vicente", "Santos"]
              }


Validação do DDD



Formato inválido

Quando consultado um DDD de formato inválido, por exemplo: "A13" (alfanumérico),"0 13" (espaço), o código de retorno da consulta será um 400 (Bad Request). Antes de acessar o webservice, valide o formato do DDD e certifique-se que o mesmo possua {2} "13" ou {3} "013" dígitos.

Exemplo de formato inválido do DDD no exemplo abaixo.

JSON


URL: viaddd.com.br/api/A13

URL: viaddd.com.br/api/A13 
            
              {
                "erro":"Formato inválido!"
              }


Dado inexistente

Quando consultado um DDD de formato válido, porém inexistente, por exemplo: "100" ou "999", o retorno conterá um valor de "erro". Isso significa que o DDD consultado não foi encontrado na base de dados. Veja como manipular este "erro" em javascript nos exemplos abaixo.

Exemplo de formato inválido do DDD no exemplo abaixo.

JSON


URL: viaddd.com.br/api/999 

            
              {
                "erro":"DDD informado não encontrado na base de dados!"
              }
